1. http://localhost:5000/api/v1/auth/signup          = for signup, uniqe email, role- costomer or admin, put every value name, email - uniqe, password, phone, role

2. http://localhost:5000/api/v1/auth/signin          = for signin, put email and password must

3. http://localhost:5000/api/v1/vehicles             = for create vehicle, admin only, put every value vehicle_name, type, registration_number - uniqe, daily_rent_price, availability_status 

4. http://localhost:5000/api/v1/vehicles             = for get all vehicles for public

5. http://localhost:5000/api/v1/vehicles/:vehicleId  = for get single vehicle for public

6. http://localhost:5000/api/v1/vehicles/:vehicleId  = for update single vehicle for admin only, put all value vehicle_name, type, registration_number - uniqe, daily_rent_price, availability_status 

7. http://localhost:5000/api/v1/vehicles/:vehicleId  = for delete single vehicle for admin only

8. http://localhost:5000/api/v1/users                = for get all users for admin only

9. http://localhost:5000/api/v1/users/:userId        = for update users, admin can update users role also details, customer can update own profile not his own role, admin can change users role customer to admin
                                                       when admin change costomer role to admin, customer need to login agin for get new token, for access full system

10. http://localhost:5000/api/v1/users/userId        = for delete user, for admin only but if customer have active booking admin can not delete this user
 
11. http://localhost:5000/api/v1/bookings            = for create booking, for customer or admin. When you create a booking you must put token customer_id in request body.

12. http://localhost:5000/api/v1/bookings            = for Get all bookings, Role-based (Admin sees all, Customer sees own)

13. admin view = http://localhost:5000/api/v1/bookings//:bookingId     = for change status in bookings, admin can change any customers booking status acative to returned..

13. customer view = http://localhost:5000/api/v1/bookings//:bookingId  = for change status in bookings, if change active to cancel then same time update vehicle status booked to available. when you update bookings status you must put valid token, which one create bookings.
